Web Hosting

Hosting a website on Amazon Web Services (AWS)

Although I see a lot of web applications made during hackathons, I see a fewer number that’s actually hosted on the web. Understanding the hosting process can be difficult for many hackers, so I decided to go over the basics of making a Node web app accessible on the world wide web. This link is less about a project than the other ones, and more focused on providing a guided run-through of how one might get their website up and running quickly. For this project, I will be using the Express.js framework downloaded through the npm package manager on Node. If you would like to know more about Node, I would suggest checking out this Node presentation I made for a past presentation. You can download the presentation here: Randy’s Node Presentation Slides. More resources will also be provided below.

Node

Node.js is one of many software alternatives that allow a computer to give a user files when they enter a url in their browser. In this project, we will be using it to give the desired files to the user who enters the url in the browser.

AWS

As defined by Google, “Amazon Web Services is a subsidiary of Amazon.com that provides on-demand cloud computing platforms to individuals, companies and governments.” As said by a former professor, cloud computing is just a fancy way to say “a computer elsewhere doing work for you.” This means that cloud services basically just give you access to computing resources (that might be too expensive or tedious to build yourself) over the internet. In this case, we’ll be requesting an AWS ‘instance’ (which is a server computer provided by Amazon servers over the internet).

HTML/CSS/Javascript

This tutorial project will require knowledge of the HTML and CSS markup languages, and beginner level understanding of Javascript. In this project, we’ll be focusing solely on the front-end as a back-end isn’t necessary for the purposes of demonstrating how to host a website.

Project Implementation

  1. The first thing I’m going to need you to do, is find a Node project to use. If you can't find one, fork and clone this link to a folder on your computer that you're comfortable with.
  2. Now what I’m going to need you to do is create a github project and commit and push your Node project into the repository . This will be the premade project referenced in the previous step or the pre-existing Node project that you made from scratch.
  3. Next, we’re going to go to the AWS website , and start a new ‘instance’. Don’t forget to save your ‘.pem’ file!!
  4. Now we’re going to need to SSH into our server and install Node (this part might get kind of tedious)
  5. Now that Node is installed, we can deploy our project! To start, clone your repository into the server you SSH’ed into.
  6. Now cd into the folder you have your content cloned into.
  7. Finally we run your project. Usually, you would type ‘node’ and your server file name to start the program, but this time you’re going to type ‘nohup’ followed by your server file name and a space and ampersand. Exit out of your server by typing ‘exit’. You’re site has now technically been launched. Now we just have to do a few things on the AWS configuration side to figure the rest of the stuff out.
  8. Go to the EC2 console and select your security group
  9. Change inbound connections to allow port 80 Find the IP address of your server You’re good to go! Now your website can be viewed from the web!
